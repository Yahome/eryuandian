# TWA-001 前置架构说明汇总

## 结论

用户已确认采纳 deep research 技术路线。TWA-001 的技术栈冻结结论调整为：Web 主线使用 Next.js 16 + React 19，后端使用 Node.js 24 LTS + NestJS + Fastify adapter，数据与任务层使用 PostgreSQL + Drizzle ORM + Redis + BullMQ，工程底座使用 pnpm workspace + Turborepo + Biome + Zod + Storybook + MSW + Playwright。

本轮仍只做技术路线冻结和事实源同步：不实现 UI、不安装依赖、不创建真实 API、不修改数据库 schema、不接短信 / 模型 / 支付。

## 冻结技术栈

| 层 | 冻结选择 |
|---|---|
| Web 前端 | Next.js 16 + React 19 + TypeScript + Tailwind CSS 4 + shadcn/ui + TanStack Query |
| API 后端 | Node.js 24 LTS + NestJS + Fastify adapter + OpenAPI |
| 数据与任务 | PostgreSQL 17/18 + Drizzle ORM + Redis + BullMQ |
| AI 接口层 | provider adapter + Responses API + Structured Outputs |
| 工程化 | pnpm workspace + Turborepo + Biome + Zod + Storybook + MSW + Playwright |
| 移动端后续 | Expo + React Native + Expo Router + NativeWind，独立 UI，共享领域层 |

## 采纳原因

- 更适合 AI 协同开发：有明确目录约定、版本匹配文档、AGENTS.md、MCP / LLM 文档入口和自动化验证链路。
- 设计稿体现的是表单驱动的生成型业务台，需要成熟的表单、状态、上传、预览、生成任务、额度、导出和移动端差异化导航能力。
- Next.js App Router 更适合作为长期 Web 主线，Vite SPA 降为备选。
- NestJS + Fastify adapter 提供更稳定的模块、控制器、服务、DI、验证、OpenAPI 组织，对 AI 更可预测。
- PostgreSQL + Drizzle 仍是主数据库与 ORM 方向。
- Redis + BullMQ 作为生成任务队列目标栈，但不在本轮落地。
- Web 先行，移动端后续用 Expo / React Native 独立 app 线；共享 contract、API client、auth、design token 和 domain logic，不强求共享复杂 UI。

## 下一步实施原则
