# PRODUCT_REQUIREMENTS：产品需求

状态：active
更新时间：2026-05-02

## 一句话定位

两元店是一个让本地商家、老师、家长、学生快速生成可用内容的 AI Web App，首期聚焦“营销生图”和“智能试卷生成”。

## 目标用户

### 本地商家

典型用户：餐饮店、零售店、美业店、培训机构、小区周边服务商。

核心诉求：

- 快速做活动图、宣传图、节日图、商品主图。
- 不懂设计也能生成看起来专业的图片。
- 能复用模板和历史作品。

### 老师 / 家长 / 学生

核心诉求：

- 根据年级、学科、教材版本、单元或考试类型快速生成试卷。
- 支持难度、题量、题型配置。
- 能预览并导出 Word / PDF。

## 首期功能

### 账号与登录

- 手机号验证码登录 / 注册。
- 密码登录可作为扩展入口。
- 用户协议勾选。
- 登录后进入首页工作台。

### 首页工作台

PC 端：

- 欢迎区。
- 生图入口。
- 试卷生成入口。
- 数据统计卡片。
- 快速开始步骤。
- 最近创作。
- 推荐模板。
- 会员权益提示。

移动端：

- 顶部品牌与用户信息。
- 两大核心功能入口。
- 快速开始流程。
- 最近创作。
- 推荐模板。

### 商家营销生图

输入：

- 图片比例。
- 业务场景。
- 行业类目。
- 需求描述 / Prompt。
- 商品图上传。
- 风格选择。

输出：

- 多张生成结果。
- 预览。
- 保存到生成记录。
- 后续支持下载、再编辑、复用模板。

### 智能生成试卷

输入：

- 年级。
- 科目。
- 教材版本，例如人教版、湘教版。
- 单元 / 范围。
- 考试类型，例如单元测试、期中、期末。
- 难度。
- 题量。
- 题型。

输出：

- 试卷预览。
- 答案与解析预留。
- Word / PDF 导出。
- 保存到生成记录。

### 生成记录

- 记录生图结果。
- 记录试卷结果。
- 支持查看、复用、删除。

### 会员与额度

首期只做基础模型和展示预留：

- 用户当前额度。
- 今日使用次数。
- 会员权益提示。
- 扣减规则先文档化，后续再接支付。

### Dev OS Dashboard

Dashboard 是开发过程可视化，不是业务后台。

展示：

- 整体进度。
- 任务看板。
- Agent 状态。
- Roadmap。
- 风险与决策。
- Wiki 快速入口。

数据来源：

- `docs/project-os/*.md`
- `docs/project-os/dashboard.json`

禁止：

- 硬编码示例统计数字。

## 非功能需求

- PC / 移动端都必须可用。
- UI 必须严格参考 `/root/eryuandian/pic`。
- 代码结构要支持前后端分离。
- API 错误返回要清晰。
- 用户输入必须做校验。
- 密钥不得进入代码仓库。

## 首期成功标准

- 用户能完成手机号登录。
- 用户能进入首页工作台。
- 用户能发起一次生图请求并看到结果。
- 用户能配置一次试卷并看到预览。
- 用户能查看生成记录。
- Dashboard 能读取真实项目事实源并展示进度。
